## 2. Модуль генерации данных (`src/data_generator.py`)

Данный модуль отвечает за создание синтетических двумерных пространств признаков для проверки обобщающей способности линейных моделей.

### Функция `get_sklearn_data()`
Служит интерфейсом к генератору `scikit-learn` для создания базового датасета.
* **Алгоритм:** Вызывает функцию `make_classification`. Генерирует 500 точек, распределенных по двум классам. Параметры фиксируют отсутствие избыточных признаков (`n_redundant=0`) и обеспечивают группировку каждого класса в один компактный кластер.

### Класс `CustomDataGenerator`
Реализует математические методы генерации линейно и нелинейно разделимых множеств.

#### Метод `generate_linear_data()`
* **Логика работы:** Генерирует два двумерных облака точек с нормальным распределением. Центры облаков задаются векторами математических ожиданий `mean1` и `mean2`. Разброс и форма облаков определяются матрицей ковариации `cov`.
* **Математический аппарат:** Для каждого класса генерируется массив координат с помощью многомерного нормального распределения:
  $$X \sim \mathcal{N}(\mu, \Sigma)$$
  Полученные матрицы объединяются по вертикали (`np.vstack`), а метки классов ($0$ и $1$) — по горизонтали (`np.hstack`).

#### Метод `generate_xor_data()`
* **Логика работы:** Генерирует нелинейно разделимое распределение типа "исключающее ИЛИ".
* **Математический аппарат:** Координаты точек случайно распределяются в квадрате $[-2, 2] \times [-2, 2]$ с помощью равномерного распределения. Метка класса определяется знаком произведения координат:
  $$y_i = 1 \iff x_{i1} \cdot x_{i2} < 0$$
  Это разделяет плоскость крест-накрест на 4 квадранта.

#### Метод `generate_circle_data()`
* **Логика работы:** Генерирует нелинейное распределение, где один класс вложен внутрь другого.
* **Математический аппарат:** Точки случайно распределяются на плоскости. Метка класса определяется расстоянием до начала координат:
  $$y_i = 1 \iff x_{i1}^2 + x_{i2}^2 < R^2$$
  Где $R$ — заданный радиус границы круга.

#### Метод `_add_noise()`
* **Логика работы:** Вносит контролируемые искажения в метки классов для имитации зашумленной среды.
* **Математический аппарат:** Создает булеву маску, где значение `True` генерируется с вероятностью `noise_prob`. Метки классов инвертируются с помощью побитовой операции исключающего ИЛИ:
  $$y_{\text{new}} = y \oplus \text{mask}$$

## 3. Модуль предварительной обработки данных (`src/preprocessing.py`)

Модуль содержит инструменты масштабирования признаков и стратификации выборок перед подачей в нейросеть.

### Класс `StandardScaler`
Реализует Z-score стандартизацию признаков.

#### Метод `fit()`
* **Логика работы:** Вычисляет статистические параметры обучающей выборки.
* **Математический аппарат:** Находит вектор средних значений $\mu$ (`np.mean`) и вектор стандартных отклонений $\sigma$ (`np.std`) по каждому признаку (вдоль нулевой оси матрицы). Для предотвращения деления на ноль в случае константных признаков к нулевым элементам вектора $\sigma$ прибавляется малое число $\epsilon = 10^{-8}$.

#### Метод `transform()`
* **Математический аппарат:** Производит центрирование и нормирование матрицы признаков:
  $$X' = \frac{X - \mu}{\sigma}$$

#### Метод `fit_transform()`
* **Логика работы:** Последовательно выполняет шаги `fit` и `transform`. Используется исключительно на обучающей выборке во избежание утечки данных.

### Функция `train_test_split_stratified()`
* **Логика работы:** Разделяет матрицы признаков и векторов меток на тренировочную и валидационную выборки в заданном соотношении с сохранением баланса классов.
* **Алгоритм:**
  1. Выделяет уникальные метки классов.
  2. Для каждого класса отдельно находит индексы элементов, перемешивает их случайным образом.
  3. Отбирает заданную долю элементов (`test_size`) от текущего класса в тестовый набор, а остальные оставляет в тренировочном.
  4. Объединяет индексы всех классов обратно и повторно перемешивает их, чтобы предотвратить группировку объектов по классам внутри выборок.

## 4. Модуль математических вычислений и метрик (`src/metrics.py`)

Содержит численные реализации функций потерь и аналитических метрик качества.

### Функция `compute_bce_loss()`
* **Математический аппарат:** Вычисляет среднее значение бинарной кросс-энтропии:
  $$L = -\frac{1}{m} \sum_{i=1}^{m} \left[ y_i \ln(\hat{y}_i) + (1 - y_i) \ln(1 - \hat{y}_i) \right]$$
* **Алгоритм:** Для защиты от неопределенности $\ln(0)$ вектор предсказаний принудительно ограничивается в диапазоне $[\epsilon, 1 - \epsilon]$, где $\epsilon = 10^{-15}$ (`np.clip`).

### Функция `accuracy_score()`
* **Математический аппарат:** Вычисляет отношение числа верных предсказаний к общему объему выборки:
  $$\text{Accuracy} = \frac{1}{m} \sum_{i=1}^{m} \mathbb{I}(y_i == \hat{y}_i)$$

### Функция `confusion_matrix()`
* **Логика работы:** Находит абсолютные значения истинно положительных (TP), ложноположительных (FP), истинно отрицательных (TN) и ложноотрицательных (FN) исходов.
* **Алгоритм:** Преобразует метки к единому бинарному виду $\{0, 1\}$. Выполняет логические операции побитового умножения (`&`) над масками совпадений истинных и предсказанных значений.

### Функции `precision_score()`, `recall_score()`, `f1_score()`
* **Математический аппарат:** Вычисляют классические метрики на основе матрицы ошибок:
  $$\text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}}, \quad \text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}}, \quad \text{F1} = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$$

### Функция `roc_curve_data()`
* **Логика работы:** Вычисляет массивы значений FPR и TPR для различных порогов принятия решений.
* **Алгоритм:** Сортирует предсказанные вероятности объектов по убыванию. Итеративно снижает порог классификации, переводя один объект за другим в класс 1, и на каждом шаге вычисляет доли ложноположительных (FPR) и истинно положительных (TPR) ответов.

### Функция `roc_auc_score()`
* **Математический аппарат:** Вычисляет площадь под ROC-кривой численным интегрированием по методу трапеций:
  $$\text{AUC} = \sum_{i=1}^{n-1} \frac{\text{TPR}_i + \text{TPR}_{i+1}}{2} \cdot (\text{FPR}_{i+1} - \text{FPR}_i)$$
* **Алгоритм:** Сортирует FPR по возрастанию для обеспечения корректности знака шага интегрирования $dx$ и суммирует площади элементарных трапеций.

## 5. Модуль нейросети (`src/perceptron.py`)

Описывает класс `Perceptron`, инкапсулирующий логику весов, прямого прохода и градиентного спуска.

### Метод `__init__()`
* **Логика работы:** Конфигурирует архитектуру модели. Задает тип функции потерь (`loss_fn`), коэффициент регуляризации `l2_lambda`. Инициализирует веса $w$ случайными величинами со средним $0$ и масштабом $0.01$ в виде вектора-столбца размера `(input_size, 1)`. Инициализирует смещение $b$ нулем.

### Метод `sigmoid()`
* **Математический аппарат:** Вычисляет логистическую функцию активации:
  $$\sigma(z) = \frac{1}{1 + e^{-z}}$$
* **Алгоритм:** Ограничивает вектор аргументов диапазоном $[-500, 500]$ (`np.clip`) для предотвращения арифметического переполнения при расчете экспоненты.

### Метод `forward()`
* **Математический аппарат:** Производит прямой проход по сети:
  $$z = X \cdot w + b$$
  Для BCE возвращает $\sigma(z)$, для Hinge Loss возвращает непосредственно линейный выход $z$.

### Метод `predict()`
* **Логика работы:** Осуществляет дискретную классификацию.
* **Алгоритм:** Для BCE разделяющий порог равен $0.5$ (выходные метки $\{0, 1\}$). Для Hinge Loss разделяющий порог равен $0.0$ (выходные метки $\{-1, 1\}$).

### Метод `_compute_l2_loss()`
* **Математический аппарат:** Вычисляет квадратичный штраф регуляризации:
  $$E_{\text{L2}} = \frac{\lambda}{2} \sum_{j=1}^{n} w_j^2$$

### Метод `fit()`
Реализует процесс градиентной оптимизации параметров перцептрона.

#### Алгоритм оптимизации (по шагам):
1. **Преобразование меток:** В случае Hinge Loss переводит метки обучающей выборки из формата $\{0, 1\}$ в формат $\{-1, 1\}$.
2. **Инициализация скоростей:** Зануляет тензоры инерции скорости весов $v_w$ и смещения $v_b$ для оптимизатора Momentum.
3. **Эпохальный цикл:**
   * Случайным образом переставляет индексы всей обучающей выборки перед началом эпохи (`np.random.permutation`).
   * **Батч-цикл:** Нарезает выборку на последовательные батчи заданного размера `batch_size`.
     * **Прямой ход:** Вызывает `forward(X_batch)`.
     * **Вычисление базовых градиентов:**
       * **BCE:**
         $$dz = \hat{y} - y$$
         $$dw = \frac{1}{m_b} X^T \cdot dz, \quad db = \frac{1}{m_b} \sum dz$$
       * **Hinge Loss:** Находит объекты, нарушившие границу ($y_i \cdot z_i < 1$). Вычисляет субградиент только по подмножеству ошибочных объектов:
         $$dw = -\frac{1}{m_{\text{err}}} X_{\text{err}}^T \cdot y_{\text{err}}, \quad db = -\frac{1}{m_{\text{err}}} \sum y_{\text{err}}$$
     * **Добавление L2-регуляризации:**
       $$dw = dw + \lambda \cdot w$$
     * **Применение Momentum SGD и обновление параметров:**
       $$v_w = \beta \cdot v_w + dw, \quad v_b = \beta \cdot v_b + db$$
       $$w = w - \eta \cdot v_w, \quad b = b - \eta \cdot v_b$$
4. **Сбор статистики:** В конце каждой эпохи вычисляет полные потери на обучающем и валидационном наборах и сохраняет их в словарь `self.history`.

## 6. Модуль визуализации (`src/visualization.py`)

Обеспечивает графическое представление процессов обучения и валидации модели с использованием библиотеки `matplotlib`.

### Функция `plot_learning_curve()`
Строит двумерный линейный график зависимости функции потерь (ось Y) от номера эпохи обучения (ось X) параллельно для обучающего и валидационного множеств.

### Функция `plot_decision_boundary()`
Отображает границу принятия решений модели на плоскости.
* **Алгоритм построения аналитической линии:**
  Разделяющая прямая задается уравнением:
  $$w_1 x_1 + w_2 x_2 + b = 0 \quad \Rightarrow \quad x_2 = -\frac{w_1 x_1 + b}{w_2}$$
  Функция вычисляет крайние значения $x_2$ для границ графика по оси X и строит зеленую линию $x_2(x_1)$.
* **Алгоритм построения фоновой заливки:**
  Создает регулярную сетку координат `np.meshgrid` с разрешением $200 \times 200$ точек в диапазоне распределения данных. Пропускает координаты сетки через метод `predict` перцептрона. Окрашивает области плоскости бледным цветом соответствующего класса с помощью метода `plt.contourf`.

### Функция `plot_roc_curve()`
Строит график ROC-кривой в осях FPR (ось X) и TPR (ось Y). Отображает диагональную пунктирную линию, соответствующую случайному угадыванию (AUC = 0.5), и выводит рассчитанную площадь под кривой (AUC) в легенду графика.

### Функция `plot_misclassified()`
Позволяет локализовать слабые стороны классификатора. На основе сравнения векторов предсказаний и истинных меток делит точки на две группы. Верные ответы отрисовываются полупрозрачными кругами синего и зеленого цветов, а неверные ответы выделяются крупными ярко-красными маркерами в форме крестов `X`.
